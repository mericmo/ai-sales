2. 数据加密

传输加密

所有数据交互强制使用 TLS 1.3 协议，禁用弱密码套件。API 端点支持 HTTP over TLS，提供完美的前向保密（PFS）。

存储加密

数据库、对象存储均采用 AES-256 透明加密。加密密钥托管于云 KMS，支持定期轮换。客户可申请自带密钥（BYOK）企业版功能。

API 密钥保护

API 密钥使用 HSM（硬件安全模块）存储，任何人无法以明文形式导出。密钥泄露可实时吊销。

3. 访问控制与身份认证

最小权限原则

每位员工仅被授予完成本职工作所需的最小数据权限。生产环境访问需通过堡垒机 + 动态令牌双重认证。

多因素认证（MFA）

所有管理后台、云控制台强制启用 MFA。客户可选择为子账户开启 MFA 增强安全。

审计日志

所有敏感操作（数据访问、权限变更、API密钥生成）均记录审计日志，保存至少180天，不可篡改。

网络隔离

生产环境与开发/测试环境严格 VPC 隔离，客户数据不跨租户共享。

4. 基础设施与容灾

云服务商：所有服务部署于阿里云/腾讯云金融级数据中心，通过等保三级认证，提供 99.995% 的物理环境可用性承诺。

多可用区部署：核心服务跨 3 个可用区（Availability Zone）部署，任一可用区故障不影响整体服务。

数据备份：客户数据每日自动备份，保留 30 天。支持按时间点恢复（PITR），RPO ≤ 15分钟，RTO ≤ 2小时。

灾难恢复演练每季度执行一次，确保极端情况下业务连续性。

7. 数据隐私保护

数据最小化 — 仅收集提供服务所必需的数据，不过度采集。

数据脱敏 — 在开发、测试、算法训练环境中，客户数据经过匿名化/脱敏处理，无法关联到具体企业。

数据保留 — 严格遵守数据保留期限，到期后自动删除或匿名化处理。客户可随时要求删除数据。

数据主权 — 所有数据存储在中国境内，不进行跨境传输。如有特殊需求，需经客户书面授权并签署数据跨境协议。

8. 安全事件应急响应

8.1 建立 7x24 小时安全监控与应急响应团队（SIRT），实时检测异常行为。

8.2 安全事件分级响应：

  • P0级（重大泄露） — 15分钟内启动应急，1小时内通知受影响客户，4小时内完成遏制。
